Frequently Asked Questions

How does GraphLoom prevent LLM hallucinations? +
GraphLoom prevents hallucinations by providing verified context from the actual codebase. Instead of relying on the LLM's training data, we extract precise subgraphs from the knowledge graph that contain factual information about the code structure, relationships, and implementations. This context acts as a ground truth that constrains the LLM's responses to accurate information.
What programming languages are supported? +
Currently, GraphLoom has robust support for Python with full AST parsing capabilities. The tree-sitter framework allows for easy extension to other languages including JavaScript, TypeScript, Java, C++, Go, and Rust. Language support is modular, allowing you to add custom parsers for domain-specific languages.
Can I use my own LLM models? +
Yes, GraphLoom provides a flexible LLM interface that supports custom model integration. Beyond the built-in support for OpenAI and Llama 3, you can implement custom adapters for any LLM that accepts text prompts. The system handles context injection and response parsing, making it straightforward to integrate proprietary or fine-tuned models.
What are the system requirements? +
Minimum requirements: Python 3.8+, 8GB RAM, 10GB disk space. Recommended: 16GB+ RAM for large codebases, SSD storage for GraphDB, GPU for local LLM inference (optional). Docker is required for the GraphDB container. The system runs on Linux, macOS, and Windows (WSL2 recommended).
